## On the battle for the Human Interface

I watched this video (https://www.youtube.com/watch?v=DhqEDFzI-OM), and it was very
much, in my eyes, a masterclass on how Linear will be the main human interface for agents, not the other way around. 


It seems that Anthropic (and AI labs), really want to be the Human Interfae into the models. ChatGPT did so well, so many people use that because of it's hold on the consumer market, and yet, there is a not an agreeded upon interface for agents. Sure you can say Claude Code (I don't use Claude Code, used to but mid), but when agents are autonommous, are you going ot be the one driving them rather than them picking up tickets?

I think it will be the latter.

So why is Linear winning?


Everyone will insert `taste` as the answer, and I agree to some degree, it's a nice product sure. I think a kanban board could achieved by any competent engineer and Claude nowadays. What makes Linear win? Especially against a model provider with an interface like Claude?

## Using Claude

So I have tried a lot of these model interfaces, I've used Claude, I've used OpenClaw, I've used DenchClaw, I've used Hermes, I've used OpenCode incredibly retrofitted (honestly not as bad as you think). The issue with these is that I believe they arguably try to be a middle ground between model and human interaction. They almost allow both as a way to appeal to both sides of the aisle rather than take a stand.


For any alteration of OpenClaw-esque agents, the abstraction becomes the issue when the models don't work.
